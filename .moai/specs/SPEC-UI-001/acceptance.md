# SPEC-UI-001 인수 기준

## 1. 캔버스 시스템 (Canvas System)

### AC-CANVAS-001: 무한 캔버스 pan/zoom 동작

**Given** 사용자가 StudioX 애플리케이션을 열고 빈 캔버스가 표시된 상태에서
**When** 마우스 휠을 위로 스크롤하면
**Then** 캔버스가 커서 위치를 기준으로 줌 인되고, panZoom.k 값이 증가한다

**Given** 사용자가 캔버스에 여러 노드가 배치된 상태에서
**When** 캔버스 빈 영역을 클릭하고 드래그하면
**Then** 캔버스 전체가 드래그 방향으로 이동하고, panZoom.x/y 값이 갱신된다

### AC-CANVAS-002: 줌 컨트롤 동작

**Given** 캔버스가 100% 줌 레벨(k=1.0)인 상태에서
**When** ZoomControls의 줌 인(+) 버튼을 클릭하면
**Then** 캔버스 줌이 다음 단계(150%)로 증가하고, 모든 노드와 커넥션이 올바르게 확대 표시된다

**Given** 캔버스 줌이 200%인 상태에서
**When** "Fit to Screen" 버튼을 클릭하면
**Then** 모든 노드가 화면에 맞도록 줌 레벨과 위치가 자동 조정된다

### AC-CANVAS-003: 캔버스 상태 저장/복원

**Given** 사용자가 캔버스를 특정 위치와 줌 레벨로 조정한 상태에서
**When** 프로젝트를 저장하고 다시 로드하면
**Then** 캔버스의 panZoom 상태(x, y, k)가 정확히 복원된다

## 2. 노드 아키텍처 (Node Architecture)

### AC-NODE-001: 노드 생성 및 기본 속성

**Given** 사용자가 빈 캔버스를 보고 있는 상태에서
**When** Toolbar에서 "Image" 노드 타입을 선택하여 추가하면
**Then** 캔버스 뷰포트 중앙에 Image 노드가 생성되고, 기본 크기(250x350 이상), 고유 ID, Image 타입 헤더 아이콘(fuchsia), 입력 커넥터(Text, Image), 출력 커넥터(Image)가 표시된다

**Given** 사용자가 빈 캔버스를 보고 있는 상태에서
**When** Toolbar에서 "Text" 노드를 추가하면
**Then** Text 노드가 생성되고, 텍스트 입력 영역이 표시되며, 출력 커넥터(Text 타입, sky-500 색상)가 노드 우측에 표시된다

### AC-NODE-002: 노드 드래그 이동

**Given** 캔버스에 3개의 노드가 배치된 상태에서
**When** 하나의 노드 헤더(.node-drag-handle)를 드래그하면
**Then** 해당 노드만 마우스 이동에 따라 위치가 변경되고, 연결된 커넥션 라인이 실시간으로 갱신된다

**Given** Shift+Click으로 2개의 노드가 다중 선택된 상태에서
**When** 선택된 노드 중 하나를 드래그하면
**Then** 모든 선택된 노드가 동일한 거리만큼 동시에 이동한다

### AC-NODE-003: 노드 삭제 및 복제

**Given** 하나의 노드가 선택되고, 다른 노드와 커넥션이 연결된 상태에서
**When** Delete 키를 누르면
**Then** 선택된 노드와 해당 노드에 연결된 모든 커넥션이 삭제되고, 다른 노드와 커넥션은 영향받지 않는다

**Given** 데이터가 입력된 Text 노드가 선택된 상태에서
**When** 복제(Duplicate) 버튼을 클릭하면
**Then** 동일한 텍스트 데이터를 가진 새 노드가 원본 노드 옆(오프셋 30px)에 생성되고, 새로운 고유 ID를 부여받는다

### AC-NODE-004: 노드 Collapse/Bypass

**Given** Image 노드가 펼쳐진 상태에서
**When** Collapse 버튼을 클릭하면
**Then** 노드가 헤더만 표시되는 접힌 상태로 전환되고, 커넥터는 여전히 표시되어 커넥션이 유지된다

**Given** Image 노드가 정상 상태인 경우
**When** Bypass 토글을 활성화하면
**Then** 노드가 반투명(opacity 50%, grayscale)으로 표시되고, 해당 노드의 출력이 다운스트림 노드에 전달되지 않는다

### AC-NODE-005: 노드 리사이즈

**Given** 기본 크기의 노드가 있는 상태에서
**When** 노드 우하단의 리사이즈 핸들을 드래그하면
**Then** 노드 크기가 드래그 방향으로 조절되되, 최소 크기(250x150) 미만으로는 줄어들지 않는다

## 3. 커넥션 시스템 (Connection System)

### AC-CONN-001: 커넥션 생성

**Given** Text 노드의 출력 커넥터(Text 타입)와 Image 노드의 입력 커넥터(Text 타입)가 있는 상태에서
**When** Text 출력 커넥터에서 마우스를 드래그하여 Image 입력 커넥터 위에서 놓으면
**Then** 두 커넥터 간에 sky 색상의 베지어 곡선 커넥션이 생성되고, Text 노드의 텍스트 데이터가 Image 노드의 입력으로 전달된다

**Given** Image 노드의 출력 커넥터(Image 타입)와 Text 노드의 입력 커넥터(Text 타입)가 있는 상태에서
**When** Image 출력 커넥터에서 Text 입력 커넥터로 드래그하면
**Then** 커넥터 타입이 불일치하므로 커넥션이 생성되지 않는다

### AC-CONN-002: 커넥션 시각적 피드백

**Given** 출력 커넥터에서 드래그를 시작한 상태에서
**When** 마우스를 캔버스 위에서 이동하면
**Then** 출력 커넥터에서 현재 마우스 위치까지 임시 커넥션 라인이 실시간으로 그려지고, 마우스 버튼을 놓으면 호환 커넥터 위가 아닌 경우 임시 라인이 사라진다

### AC-CONN-003: 데이터 전파

**Given** Text 노드 -> Image 노드로 커넥션이 연결된 상태에서
**When** Text 노드의 텍스트를 "A beautiful sunset"으로 변경하면
**Then** Image 노드가 연결된 Text 입력으로 "A beautiful sunset"을 자동으로 수신하여 프롬프트에 반영한다

## 4. AI 통합 (AI Integration)

### AC-AI-001: 이미지 생성

**Given** Text 노드("A cat sitting on a chair") -> Image 노드가 연결된 상태에서
**When** Image 노드의 Generate 버튼을 클릭하면
**Then** 노드에 로딩 인디케이터가 표시되고, Gemini API 호출이 성공하면 생성된 이미지가 노드에 표시되며, HistoryPanel에 에셋이 자동 추가된다

**Given** AI 이미지 생성 중 네트워크 오류가 발생한 경우
**When** API 응답이 실패하면
**Then** 로딩 인디케이터가 중지되고, 사용자에게 에러 메시지가 표시되며, 재시도가 가능하다

### AC-AI-002: 비디오 생성

**Given** Image 노드(생성된 이미지 보유) -> Video 노드가 연결된 상태에서
**When** Video 노드의 Generate 버튼을 클릭하면
**Then** 연결된 이미지와 텍스트 프롬프트를 기반으로 비디오가 생성되고, 노드에 비디오 플레이어가 표시된다

### AC-AI-003: Virtual Try-On

**Given** Model 노드(모델 이미지 생성 완료) -> VTON 노드, ImageLoad 노드(의상 이미지) -> VTON 노드가 연결된 상태에서
**When** VTON 노드의 Generate 버튼을 클릭하면
**Then** 모델 이미지에 의상이 합성된 결과 이미지가 생성되어 VTON 노드에 표시된다

### AC-AI-004: 배경 제거

**Given** ImageLoad 노드(배경 있는 이미지) -> RMBG 노드가 연결된 상태에서
**When** RMBG 노드의 배경 제거를 실행하면
**Then** 배경이 제거된 이미지가 생성되고, 선택된 배경색(기본값 또는 사용자 지정)이 적용되어 표시된다

## 5. 프로젝트 관리 및 UI (Project Management and UI)

### AC-PROJ-001: 프로젝트 저장/로드

**Given** 캔버스에 5개의 노드, 3개의 커넥션, 2개의 생성된 이미지가 있는 상태에서
**When** 프로젝트를 "My Project"로 저장한 후 새 프로젝트를 생성하고, 다시 "My Project"를 로드하면
**Then** 5개의 노드(위치, 크기, 데이터 포함), 3개의 커넥션, 2개의 히스토리 에셋, 캔버스 panZoom 상태가 모두 정확히 복원된다

**Given** 프로젝트에 대용량 이미지(4K 해상도) 에셋이 포함된 상태에서
**When** 프로젝트를 저장하면
**Then** IndexedDB에 정상 저장되고, 로드 시 2초 이내에 복원된다

### AC-PROJ-002: 키보드 단축키

**Given** Image 노드가 선택된 상태에서
**When** 'B' 키를 누르면
**Then** Bypass 토글이 전환된다

**Given** 여러 노드와 커넥션이 있는 상태에서
**When** Ctrl+Z를 누르면
**Then** 직전 수행한 동작(노드 이동, 데이터 변경, 삭제 등)이 취소되고 이전 상태로 복원된다

## 6. 엣지 케이스 시나리오

### EC-001: 동일 노드 자기 참조 방지

**Given** Image 노드의 출력 커넥터에서 드래그를 시작한 상태에서
**When** 같은 노드의 입력 커넥터 위에서 마우스를 놓으면
**Then** 자기 참조 커넥션은 생성되지 않는다

### EC-002: 중복 커넥션 방지

**Given** Text 노드와 Image 노드 사이에 이미 커넥션이 존재하는 상태에서
**When** 동일한 출력-입력 커넥터 쌍으로 다시 커넥션을 시도하면
**Then** 중복 커넥션은 생성되지 않는다

### EC-003: 순환 커넥션 방지

**Given** A -> B -> C 순서로 노드가 연결된 상태에서
**When** C의 출력을 A의 입력으로 연결하려 시도하면
**Then** 순환이 감지되어 커넥션 생성이 거부된다

### EC-004: 대량 노드 성능

**Given** 캔버스에 100개 이상의 노드가 배치된 상태에서
**When** 캔버스를 pan/zoom 하면
**Then** 60fps 이상의 프레임 레이트가 유지되고, 뷰포트 밖 노드는 렌더링이 스킵된다

### EC-005: 브라우저 새로고침 시 데이터 보존

**Given** 프로젝트 작업 중 자동 저장이 실행된 후
**When** 브라우저를 새로고침하면
**Then** 마지막 자동 저장 시점의 프로젝트 상태가 복원된다

### EC-006: API 키 미설정 상태

**Given** Gemini API 키가 설정되지 않은 상태에서
**When** 사용자가 Image 노드에서 Generate를 클릭하면
**Then** API 키 설정을 안내하는 메시지가 표시되고, API 호출은 수행되지 않는다

## 7. 성능 기준

| 항목 | 기준값 | 측정 방법 |
|------|--------|----------|
| 캔버스 pan/zoom FPS | 60fps 이상 | Chrome DevTools Performance |
| 100개 노드 렌더링 | 16ms 이하/프레임 | React Profiler |
| 프로젝트 저장 | 2초 이내 | Performance.now() 측정 |
| 프로젝트 로드 | 2초 이내 | Performance.now() 측정 |
| AI 이미지 생성 응답 | 60초 이내 | API 타임아웃 설정 |
| 노드 드래그 반응 | 즉시(16ms 이내) | 입력 지연 측정 |
| 커넥션 라인 렌더링 | 1000개까지 60fps | SVG 렌더링 성능 |
| 메모리 사용량 | 500MB 이하 (100노드 기준) | Chrome Task Manager |
